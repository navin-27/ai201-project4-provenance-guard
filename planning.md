# Provenance Guard - Planning Document

## Project Overview

Provenance Guard is a backend system that helps determine whether submitted text is more likely to be written by a human or generated by AI. The system analyzes each submission using multiple detection signals, calculates a confidence score, displays a transparency label, records all decisions in an audit log, and allows creators to appeal classifications they believe are incorrect.

---

# Detection Signals

## Signal 1 – LLM-Based Classification (Groq)

**Purpose**

Uses the Groq Llama 3.3 70B model to analyze writing style, coherence, fluency, tone, and semantic consistency.

**Output**

A probability score between 0.0 and 1.0 representing the likelihood that the text is AI-generated.

**Strengths**

- Understands context
- Detects AI writing patterns
- Considers overall writing quality

**Limitations**

- May mistake polished human writing for AI.
- Depends on LLM reasoning.

---

## Signal 2 – Stylometric Analysis

This signal analyzes measurable writing characteristics without using AI.

Metrics include:

- Sentence length variation
- Vocabulary diversity (Type-Token Ratio)
- Punctuation density

Output:

A score between 0.0 and 1.0.

Strengths:

- Fast
- Explainable
- Does not require an API call

Limitations:

- Short texts may not provide enough information.
- Creative writing can resemble AI-generated text.

---

# Confidence Scoring

The final confidence score combines both signals.

Formula:

Confidence = (LLM Score × 0.6) + (Stylometric Score × 0.4)

Classification Thresholds

0.75 – 1.00

Likely AI Generated

0.45 – 0.74

Uncertain

0.00 – 0.44

Likely Human Written

The weighting favors the LLM because semantic understanding is generally more reliable than simple statistical writing features.

# Transparency Labels

## High Confidence AI

"This content is likely AI-generated. Our system detected multiple indicators associated with AI-generated writing. Confidence: High."

---

## High Confidence Human

"This content appears to be written by a human. No strong indicators of AI-generated writing were detected. Confidence: High."

---

## Uncertain

"Our system could not confidently determine whether this content was written by a human or generated by AI. Additional review may be required."

---

# Appeals Workflow

Creators may appeal any classification.

The appeal includes:

- Content ID
- Creator reasoning

System actions:

- Update status to "Under Review"
- Save appeal reason
- Log the appeal
- Keep the original classification for reviewer reference

Human reviewers can later inspect:

- Original text
- Both signal scores
- Confidence score
- Classification
- Appeal reason

---

# Anticipated Edge Cases

## Edge Case 1

Poetry with repetitive wording may resemble AI-generated text because of limited vocabulary.

---

## Edge Case 2

Highly edited AI-generated text may appear human-written and receive a lower AI confidence score.

---

## Edge Case 3

Very short submissions may not contain enough linguistic information for accurate stylometric analysis.

The system begins when a creator submits text using the POST /submit endpoint. The submitted text is analyzed independently by two detection signals: the Groq LLM detector and a stylometric analysis module. The outputs from both signals are combined into a weighted confidence score. Based on this score, the system generates one of three transparency labels: Likely AI Generated, Uncertain, or Likely Human Written.

After classification, the submission details—including the confidence score, individual signal scores, timestamp, and attribution result—are stored in a structured audit log. The response is then returned to the client. If the creator disagrees with the result, they can submit an appeal through the POST /appeal endpoint. The appeal updates the submission status to "Under Review" and is recorded in the audit log for future review.

## API Endpoints

### POST /submit

Purpose:
Accepts text content from a creator for attribution analysis.

Input

- creator_id
- text

Returns

- content_id
- attribution
- confidence
- transparency label

---

### POST /appeal

Purpose

Allows creators to challenge a classification.

Input

- content_id
- creator_reasoning

Returns

- confirmation message
- updated status

---

### GET /log

Purpose

Returns recent audit log entries for review and testing.
