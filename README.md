# ai201-project4-provenance-guard# Provenance Guard

## Overview

Provenance Guard is a Flask-based backend system that analyzes submitted text and estimates whether it was likely written by a human or generated by AI.

The system combines multiple independent detection signals, calculates a confidence score, displays a transparency label, stores every submission in an audit log, supports creator appeals, and applies rate limiting to protect the API from abuse.

---

# Features

- Multi-signal AI content detection
- Confidence scoring
- Transparency labels
- Appeals workflow
- Audit logging
- Rate limiting
- REST API built with Flask

# Detection Signals

## Signal 1 – Groq LLM

The first signal uses the Groq Llama 3.3 70B model to analyze the semantic characteristics of submitted text. It evaluates writing style, coherence, tone, fluency, and overall language patterns to estimate whether the text is AI-generated or human-written.

Strengths

- Understands context
- Recognizes AI writing patterns
- Uses semantic reasoning

Limitations

- May classify polished human writing as AI.
- Depends on LLM inference.

---

## Signal 2 – Stylometric Analysis

The second signal analyzes measurable writing characteristics without using AI.

Metrics include

- Sentence length variation
- Vocabulary diversity
- Punctuation density

Strengths

- Fast
- Explainable
- No API required

Limitations

- Short text provides limited information.
- Creative writing can resemble AI-generated text.

---

# Confidence Scoring

The final confidence score combines both signals.

Formula

```
Confidence =
(LLM Score × 0.6)
+
(Stylometric Score × 0.4)
```

Example 1

Groq Score

```
0.85
```

Stylometric Score

```
0.51
```

Combined

```
0.71
```

Label

```
Uncertain
```

Example 2

Groq Score

```
0.94
```

Stylometric Score

```
0.80
```

Combined

```
0.88
```

Label

```
Likely AI Generated
```

---

# Appeals Workflow

Creators can appeal any classification.

Each appeal includes

- Content ID
- Creator reasoning

The system

- Updates status to **under_review**
- Stores the appeal
- Preserves the original classification
- Updates the audit log

---

# Audit Log

Each submission records

- Timestamp
- Content ID
- Creator ID
- Attribution
- LLM Score
- Stylometric Score
- Confidence Score
- Label
- Status
- Appeal Reason (if submitted)

---

# Rate Limiting

The submission endpoint is protected using Flask-Limiter.

Limit

```
10 requests per minute
100 requests per day
```

Reason

These limits are appropriate for normal writers while protecting the system against automated abuse and denial-of-service attempts.

---

# Known Limitations

The current system may struggle with

- Poetry
- Song lyrics
- Very short text
- Highly edited AI-generated writing
- Formal academic writing

Perfect AI detection is an unsolved problem. The system therefore provides confidence scores and an appeal process rather than making absolute decisions.
